# Platform Philosophy

The repository is designed around three principles:

## 1. Preserve Real Template Identity

Existing template packages are treated as preserved source packages. Platform work should happen around them through metadata, registry entries, documentation, and routing contracts unless a future task explicitly targets the template itself.

## 2. Stay Honest About What Exists

The platform must clearly distinguish between:

- real templates
- scaffolds
- empty future categories

The repository should not look more populated than it is.

## 3. Optimize For Long-Term Growth

The architecture should support:

- many template categories
- consistent metadata
- registry-driven automation
- future preview pipelines
- future web-based browsing and filtering
